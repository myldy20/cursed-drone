# Android-версия: alpha 2

Android-порт сохраняет существующий C++-аудиодвижок и SDL-интерфейс. Android-слой отвечает за полноэкранный запуск, ARM64-сборку, данные приложения, профиль аудиобуфера и прямое сенсорное управление.

## Статус

Это **alpha-версия** для проверки на реальных устройствах.

- Android 8.0 и новее (`minSdk 26`)
- только ARM64 (`arm64-v8a`)
- ландшафтная ориентация
- наушники, USB-аудиоинтерфейсы и динамик телефона работают через SDL/Android Audio
- autosave и восемь слотов памяти хранятся в закрытой папке приложения

## Установка сборки из CI

1. Откройте запуск workflow `android` в GitHub Actions.
2. Скачайте артефакт `cursed-drone-android-debug`.
3. Распакуйте его и установите `app-debug.apk` на Android-устройство.
4. Android может попросить разрешить установку приложений из браузера или файлового менеджера, через который открыт APK.

Debug APK подписан стандартным отладочным ключом Android. Поэтому Play Protect может сообщать, что приложение получено от неизвестного или непроверенного разработчика. Для нормального публичного распространения потребуются постоянный release-ключ и регистрация разработчика; это не является признаком ошибки аудиодвижка.

## Сенсорное управление

Отдельной панели виртуальных кнопок больше нет. Интерфейс масштабируется целиком под доступную ландшафтную область, сохраняя пропорции исходной компоновки.

- касание вкладки сразу открывает нужную страницу;
- касание актёра, FX-слота, строки параметра или слота памяти сразу выбирает этот элемент;
- горизонтальное движение пальца по строке изменяет выбранное значение;
- касание ландшафта, движка или типа эффекта открывает существующий список;
- касание варианта во всплывающем списке сразу выбирает и применяет его;
- нижняя кнопка `FADE` плавно закрывает или открывает итоговый выход;
- системная кнопка/жест Android «Назад» закрывает список или возвращает на страницу места.

На странице места нижняя область карточки актёра управляет mute, а остальная часть карточки выбирает актёра.

Внешние клавиатуры и геймпады продолжают работать через SDL, но Android-интерфейс больше не строится вокруг их кнопок.

## Аудиопрофиль Android

Приложение запрашивает у Android нативную частоту дискретизации и размер аппаратного burst-буфера. Затем SDL открывает более устойчивый callback-буфер размером не менее 2048 кадров. Это увеличивает задержку управления звуком, но заметно снижает риск underrun и гранулированных прерываний. Для дронового инструмента стабильность важнее минимальной игровой задержки.

## Локальная сборка

Понадобятся:

- Android Studio или Android command-line tools
- JDK 17
- Android SDK 35
- Android NDK `27.1.12297006`
- CMake 3.22.1
- Gradle 8.10.2 или совместимая версия

Клонируйте репозиторий вместе с сабмодулями и соберите APK:

```bash
git clone --recurse-submodules https://github.com/myldy20/cursed-drone.git
cd cursed-drone
gradle -p android :app:assembleDebug
```

Готовый APK появится здесь:

```text
android/app/build/outputs/apk/debug/app-debug.apk
```

Во время сборки Gradle загружает официальный Android Java-shim SDL2 и нативный исходный код SDL2 из зафиксированного тега `release-2.32.10`. Аудиодвижок Cursed Drone и сабмодуль Plaits компилируются непосредственно в APK.

## Известные ограничения alpha

- собирается только ARM64;
- пока нет Android App Bundle и постоянной release-подписи;
- портретная компоновка не поддерживается: это потребует отдельного интерфейса, а не простого масштабирования;
- прямой touch-слой повторяет состояние исходного SDL-интерфейса и требует проверки всех редких комбинаций переходов;
- задержка и маршрутизация USB/Bluetooth-аудио зависят от устройства и прошивки Android;
- обработку звонков, смены аудиоустройств и горячего подключения интерфейса нужно проверить на железе.
