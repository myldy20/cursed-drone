# Разбор звукового референса

Пользовательский WAV не хранится в репозитории. Ниже зафиксированы только измерения и выводы, нужные для воспроизводимого проектирования движков.

## Измерения

Референс длится 47,426 с, записан в stereo 44,1 kHz / float32. Интегральный RMS около −12,08 dBFS, crest factor 13,55 dB, корреляция каналов 0,88. Float-файл кратковременно превышает условный 0 dBFS примерно на 1,47 dB; это не целевой уровень для движка Cursed Drone.

- 42,5% спектральной энергии находится в диапазоне 20–80 Hz;
- 47,2% — в 80–250 Hz;
- ещё 7,8% — в 250–800 Hz;
- основной устойчивый пик близок к 64,6 Hz, следующий — 129,2 Hz;
- медианный spectral centroid — около 517 Hz;
- наиболее заметный крупный период огибающей — примерно 1,33 с, или 45,2 BPM.

Это низкочастотное тело с гармониками, но его характер создаёт не статический осциллятор. Внутри есть резонансные пачки, нерегулярные атаки, смены спектральной яркости и длинное финальное затухание после примерно 38-й секунды.

## Что из этого следует для архитектуры

Одного «дронового» движка с разными формами волны недостаточно. Нужны три независимых типа поведения:

1. непрерывное тональное тело для фона и биений;
2. возбуждаемый модальный резонатор для металлических ударов, псевдо-струнных и «лопатных» событий;
3. событийный источник частиц/крупинок с ограниченно-случайными паузами.

В `0.3.0` этим ролям соответствуют `TONE`, `RESONATOR`, `GRAINLET` и `PARTICLES`. Макрос `Events` задаёт частоту и нерегулярность событий, `Pulse` — глубину и плотность ритмических огибающих, а общий tempo масштабирует оба процесса.

## Целевые пространства

### Тонкий фон + крупные рассыпающиеся шаги

- `TONE`: низкая форма/шум, умеренный drift, небольшой уровень;
- `RESONATOR`: низкая или средняя частота, большое возбуждение, умеренная яркость;
- `GRAINLET` или `PARTICLES`: тихий рассыпчатый верх атаки;
- `Events` 40–70%, `Chaos` 15–35%, небольшой `Space`.

### Импульс с подъёмом и случайной паузой

Нужен отдельный генератор формы события: attack/ramp → release → bounded random hold. `0.3.0` уже даёт нерегулярные паузы и удары, но ещё не умеет привязывать к каждому событию направленный подъём высоты. Это следующий небольшой DSP-блок, а не трекерный секвенсор.

### Низкий металл + почти фортепианный верх

- два слота `RESONATOR` с разными частотами и материалом;
- нижний — тёмный, жёсткий, с редкими сильными ударами;
- верхний — ярче, короче и тише, без постоянной мелодии;
- для случайной псевдомелодии понадобится bounded quantizer: небольшой набор интервалов, вероятность повторения и запрет слишком больших скачков.

Эти три пространства используются как критерии для будущих factory scenes. Они не должны сводиться к трём заранее записанным пресетам: макросы обязаны сохранять музыкально полезную область между крайними положениями.
