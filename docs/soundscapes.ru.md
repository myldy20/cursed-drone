# Архитектура процедурных ландшафтов

## Почему четырёх синтезаторов оказалось недостаточно

Версия 0.3 доказала работоспособность аудиографа, но не музыкальную идею. Четыре разных генератора всё равно давали четыре непрерывных сигнала с похожей логикой: высота, тембр, шумовая доля, эффект. Глобальный макрос мог одновременно поднять несколько значений, но не мог создать шаг, дверной скрип, рабочий цикл или осмысленную паузу. Для этого генератору нужны состояние, начало и конец жеста, модель возбуждения и отдельный закон больших изменений.

В 0.4 слот перестал быть только «синтезатором». В стартовом патче это **актёр сцены**:

1. фоновая среда, которая может жить постоянно;
2. движение или повторяющийся процесс;
3. редкий объект переднего плана;
4. дальний, воздушный или текстурный слой.

Экспертный экран по-прежнему открывает частоту, четыре характерных параметра, уровень, панораму, FX и модуляторы. Сценовый экран работает уровнем выше и меняет причинность звука, а не только значение одного DSP-параметра.

## Три первые сцены

| Сцена | Актёр | Процедурная модель |
| --- | --- | --- |
| Заброшенное | фон | несколько медленно расходящихся низких частот, воздушные полосы, долгий направленный pitch climb и дыхание уровня |
| Заброшенное | шаги | независимые heel/toe-огибающие, низкий удар, фильтрованный шум поверхности, чередование и большой цикл приближения/удаления |
| Заброшенное | дверь | stick-slip-последовательность микроимпульсов, короткие негармонические моды дерева и направленный squeal/glissando |
| Заброшенное | труба | шумовой поток через дрейфующие резонансы, возникающий только на сильных порывах howl и редкие короткие стуки корпуса |
| Цех | мотор | отдельные rotor harmonics, высокочастотные brushes, плавающая нагрузка и медленное изменение скорости |
| Цех | станок | цикл удара и пневматического выхлопа, короткий корпус, BPM как масштаб времени, а не нотная сетка |
| Цех | толпа | три движущиеся формантные области шума с независимыми фразовыми огибающими; это масса голосов без синтеза речи |
| Цех | металл | длинное шумовое трение через движущиеся узкие полосы плюс короткий сильно затухающий clank без «колокольного» sustain |
| Пустошь | ветер | быстрый шум, медленный случайный gust и ещё более медленный squall; тональный howl появляется только на пиках потока |
| Пустошь | птицы | случайные кластеры коротких частотно-модулированных фраз с контуром, паузами и слабой зернистой составляющей |
| Пустошь | насекомые | несколько несинхронных импульсных несущих, фазы активности колонии и узкополосный шорох |
| Пустошь | сигнал | серия импульсов с направленным повышением высоты, затем ограниченно-случайная пауза и новый цикл |

Все источники синтезируются в реальном времени. Записанных шагов, дверей, птиц или заводских сэмплов в репозитории нет.

## Пять макросов

| Макрос | Временной уровень | Примеры реального воздействия |
| --- | --- | --- |
| Материал | внутри события | поверхность шага, доля щёток и пыли, трение, яркость возбуждения, шум/тон |
| Активность | события и процессы | скорость шагов, вероятность скрипа, цикл станка, размер птичьей стаи, плотность сигналов |
| Напряжение | направленное изменение | pitch climb фона, нагрузка мотора, высота howl, диапазон металлического свипа, подъём серии импульсов |
| Дистанция | микс и память | dry/wet, feedback, хвосты и разделение переднего/дальнего плана |
| Развитие | большие циклы | скорость приближения, длительность жеста, длина серии, масштаб случайной паузы, скорость мутации среды |

Макросы проходят сглаживание. Случайность детерминирована отдельным seed каждого актёра и ограничена диапазонами, поэтому offline-render воспроизводим, а опасный feedback не может возникнуть случайно.

## Исследованные открытые источники

- [DaisySP](https://github.com/electro-smith/DaisySP) остаётся встроенной MIT-основой: oscillator, stable SVF, clocked noise, grainlet, particle и modal blocks. В новом слое особенно важен state-variable filter; orchestration и event memory принадлежат Cursed Drone.
- Открытые примеры к книге Andy Farnell [Designing Sound](https://mitp-content-server.mit.edu/books/content/sectbyfn/books_pres_0/8375/designing_sound.zip/index.html) использовались как справочник по декомпозиции: wind = несколько скоростей шума; motor = rotor + brushes + body; footsteps = последовательные фазы опоры; door = stick-slip + panel modes. Pure Data patches не копируются и не входят в сборку.
- [STK](https://github.com/thestk/stk) подтвердил жизнеспособность компактных физически информированных инструментов и управляемых noise/resonance моделей. Он не добавлен зависимостью: для первых сцен его runtime, таблицы и более крупный API не дают преимущества перед уже встроенным DaisySP subset.
- [Faust libraries](https://github.com/grame-cncm/faustlibraries) содержат готовые noise, reverb и physical-model families и остаются хорошей лабораторией. Генератор Faust пока не входит в toolchain Brick, чтобы не усложнять кросс-сборку.
- [Soundpipe](https://github.com/PaulBatchelor/Soundpipe) содержит полезный каталог компактных алгоритмов, но upstream архивирован. Это справочник, а не новая обязательная зависимость.

## Что считается успехом

Автотест проверяет конечность, лимитирование, сохранение сессии и ненулевой выход всех двенадцати актёров. Этого недостаточно для музыкальной оценки. Для каждого релиза также нужны:

- минимум 30 секунд каждой сцены без ручного вмешательства;
- узнаваемая разница спектра и временной структуры трёх сцен;
- слышимые пустоты и foreground-события, а не постоянная максимальная плотность;
- радикально отличающиеся состояния низких и высоких значений макросов;
- отсутствие случайного колокольного sustain у дерева и металла;
- измерение DSP load и десятиминутный underrun-test на TrimUI Brick.

GitHub Actions рендерит три 30-секундных WAV и публикует их как артефакт `cursed-drone-soundscape-previews` рядом со сборками macOS и PortMaster.
