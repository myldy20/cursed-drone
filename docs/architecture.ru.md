# Архитектура Cursed Drone 0.12.3

```text
Макросы места
   ↓
Актёр 1 → модуляция → FX1 → FX2 → FX3 → FX4 → level/pan ┐
Актёр 2 → модуляция → FX1 → FX2 → FX3 → FX4 → level/pan ├→ mix
Актёр 3 → модуляция → FX1 → FX2 → FX3 → FX4 → level/pan ┤
Актёр 4 → модуляция → FX1 → FX2 → FX3 → FX4 → level/pan ┘
                         mix → Master FX1..4 → DC blocker → limiter → fade
```

Аудиопоток получает полные снимки `Session` через SPSC-очередь. Схема 11 хранит четырёх актёров, отдельную частоту событий каждого актёра, Scala-строй, Euclidean-настройки, четыре строки модуляции, actor FX, Master FX и исполнительские макросы. Ручной trigger передаётся через lock-free атомарную битовую маску, а вспышка события публикуется в телеметрии и не сохраняется в сессии.

Флаг `enabled` актёра имеет приоритет. Построение рецепта ландшафта не может снова включить заглушённого актёра. Выключенный актёр продолжает подавать тишину во внутренние буферы, но его индивидуальный FX-хвост не выходит в микс. Общий хвост Master FX сохраняется.

Autosave и восемь слотов памяти — читаемые `.cdrone`-файлы. Встроенные Scala-файлы лежат в assets, пользовательские читаются из рабочей папки.
