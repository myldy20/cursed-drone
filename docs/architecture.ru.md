# Архитектура Cursed Drone 0.13.0

## Один репозиторий, одно звуковое ядро, разные интерфейсы платформ

```text
                           ┌─ SDL-интерфейс с кнопочной навигацией (Brick, Linux, macOS)
Session + каталог + DSP ───┤
                           └─ touch-first SDL-интерфейс (Android)
```

Все платформы собирают один и тот же `cursed_drone_core`, выбранные модули DaisySP, закреплённый Plaits, схему сессии, Scala, каталог эффектов и рецепты мест из корневого `CMakeLists.txt`. Файл `VERSION` — единственный источник номера версии для CMake, Gradle, интерфейса и релизных пакетов. Сами интерфейсы намеренно разные: на Brick нужна навигация фокусом с контроллера, а на Android — прямые вкладки, сенсорные слайдеры и модальные сетки.

```text
Макросы места
   ↓
Актёр 1 → модуляция → FX1 → FX2 → FX3 → FX4 → level/pan ┐
Актёр 2 → модуляция → FX1 → FX2 → FX3 → FX4 → level/pan ├→ mix
Актёр 3 → модуляция → FX1 → FX2 → FX3 → FX4 → level/pan ┤
Актёр 4 → модуляция → FX1 → FX2 → FX3 → FX4 → level/pan ┘
                         mix → Master FX1..4 → DC blocker → saturator → fade
```

Аудиопоток получает полные снимки `Session` через ограниченный SPSC-mailbox. Ручной trigger передаётся атомарной битовой маской, а вспышка события остаётся телеметрией и не записывается. Схема 11 едина для всех платформ: актёры, Event Rate, Scala, Euclidean, модуляция, actor FX, Master FX и макросы совпадают на телефоне и приставке.

## Модель производительности

- Заглушённый актёр один раз сбрасывается и полностью исключается из посэмплового цикла; после включения он стартует с чистого состояния. Хвост Master FX остаётся глобальным.
- Дорогие медленно меняющиеся коэффициенты событий, модальных резонансов и эффектов считаются на control-rate; осцилляторы, огибающие, delay и нелинейная обработка остаются audio-rate.
- Equal-power pan берётся из интерполируемой таблицы вместо двух `sqrt` на каждого актёра и каждый сэмпл.
- DC blocker рассчитывается от реальной частоты дискретизации с частотой среза 12 Гц и не выедает фундамент Sub Drone и Earth Rumble.
- Kill Silence аннулирует историю delay за постоянное время и не чистит большие буферы внутри callback.

Autosave и восемь слотов используют одинаковые читаемые `.cdrone` на всех платформах. Запись атомарная, предыдущая полная сессия сохраняется в `.bak`.
